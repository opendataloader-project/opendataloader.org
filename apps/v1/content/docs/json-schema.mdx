---
title: JSON Schema
description: Understand the layout structure emitted by OpenDataLoader PDF
---

Every conversion that includes the `json` format produces a hierarchical document describing detected elements (pages, tables, lists, captions, etc.). Use the following reference to map fields into your downstream processors.

## Root node

| Field               | Type      | Optional | Description                            |
| ------------------- | --------- | -------- | -------------------------------------- |
| `file name`         | `string`  | No       | Name of the processed PDF.             |
| `number of pages`   | `integer` | No       | Total page count.                      |
| `author`            | `string`  | No       | PDF author metadata.                   |
| `title`             | `string`  | No       | PDF title metadata.                    |
| `creation date`     | `string`  | No       | PDF creation timestamp.                |
| `modification date` | `string`  | No       | PDF modification timestamp.            |
| `kids`              | `array`   | No       | Top-level content elements (per page). |

## Common content fields

| Field          | Type      | Optional | Description                                                                                                                                                               |
| -------------- | --------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id`           | `integer` | Yes      | Unique content identifier.                                                                                                                                                |
| `level`        | `string`  | Yes      | Heading or structural level.                                                                                                                                              |
| `type`         | `string`  | No       | Element type (`footer`, `header`, `heading`, `line`, `table`, `table row`, `table cell`, `paragraph`, `list`, `list item`, `image`, `line art`, `caption`, `text block`). |
| `page number`  | `integer` | No       | Page containing the element.                                                                                                                                              |
| `bounding box` | `array`   | No       | Bounding box coordinates.                                                                                                                                                 |

## Text nodes (`caption`, `heading`, `paragraph`)

| Field        | Type     | Optional | Description     |
| ------------ | -------- | -------- | --------------- |
| `font`       | `string` | No       | Font name.      |
| `font size`  | `double` | No       | Font size.      |
| `text color` | `array`  | No       | RGB color.      |
| `content`    | `string` | No       | Raw text value. |

## Tables

| Field               | Type      | Optional | Description                                       |
| ------------------- | --------- | -------- | ------------------------------------------------- |
| `number of rows`    | `integer` | No       | Row count.                                        |
| `number of columns` | `integer` | No       | Column count.                                     |
| `rows`              | `array`   | No       | Row objects.                                      |
| `previous table id` | `integer` | Yes      | Linked table identifier (if broken across pages). |
| `next table id`     | `integer` | Yes      | Linked table identifier.                          |

### Table rows

| Field        | Type      | Optional | Description   |
| ------------ | --------- | -------- | ------------- |
| `row number` | `integer` | No       | Row index.    |
| `cells`      | `array`   | No       | Cell objects. |

### Table cells

| Field           | Type      | Optional | Description                |
| --------------- | --------- | -------- | -------------------------- |
| `row number`    | `integer` | No       | Row index of the cell.     |
| `column number` | `integer` | No       | Column index of the cell.  |
| `row span`      | `integer` | No       | Number of rows spanned.    |
| `column span`   | `integer` | No       | Number of columns spanned. |
| `kids`          | `array`   | No       | Nested content elements.   |

## Headings

| Field           | Type      | Optional | Description               |
| --------------- | --------- | -------- | ------------------------- |
| `heading level` | `integer` | No       | Level (e.g., 1 for `h1`). |

## Lists

| Field                  | Type      | Optional | Description                           |
| ---------------------- | --------- | -------- | ------------------------------------- |
| `number of list items` | `integer` | No       | Item count.                           |
| `numbering style`      | `string`  | No       | Marker style (ordered, bullet, etc.). |
| `previous list id`     | `integer` | Yes      | Linked list identifier.               |
| `next list id`         | `integer` | Yes      | Linked list identifier.               |
| `list items`           | `array`   | No       | Item nodes.                           |

### List items

| Field  | Type    | Optional | Description              |
| ------ | ------- | -------- | ------------------------ |
| `kids` | `array` | No       | Nested content elements. |

## Headers and footers

| Field  | Type    | Optional | Description                                   |
| ------ | ------- | -------- | --------------------------------------------- |
| `kids` | `array` | No       | Content elements within the header or footer. |

## Text blocks

| Field  | Type    | Optional | Description          |
| ------ | ------- | -------- | -------------------- |
| `kids` | `array` | No       | Text block children. |

Use these tables as guidance for building downstream schemas or validations; the actual JSON output is designed to be self-descriptive and can be inspected with typical JSON tooling.
